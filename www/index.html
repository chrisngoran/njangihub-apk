Browser storage Device only
By default, data saves in this browser only. Turn on Cloud sync in Settings so every device sharing a group code sees the same live data.
Setting up cloud sync (one-time, for your developer)
1. Create a free project at supabase.com
2. In the SQL editor, run:
create table njangi_groups ( group_code text primary key, data jsonb, updated_at timestamptz default now() ); alter table njangi_groups enable row level security; create policy "anon read/write by code" on njangi_groups for all using (true) with check (true); 3. Copy the Project URL and anon public key from Settings → API in Supabase
4. Paste both into this app's Settings → Cloud sync, pick a unique group code, click Connect
5. Do the same on every other device this group uses — same URL, key, and group code
The RLS policy above allows anyone with the anon key to read/write any group's row. Fine for a pilot; for production, scope the policy to authenticated users or add per-group secrets.
